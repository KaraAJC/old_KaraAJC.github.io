Understanding Hashes and Arrays in Ruby
October 26, 2014

In the Ruby Programming Language, the code is represented by two separate yet equally important collections: the Hash, which pair Keys with Values; and the Array, which orders items from zero. These are their stories.

Dun Dun DUN!!

This week we got into Ruby; a programming language that we will primarily be using during Dev Bootcamp. It's quite a feat to get into, but a lot of the features of the language make sense when you spend time playing around with it.

Two key components we learned were called Hashes and Arrays. These are two very similar, but distinct ways of collecting information using Ruby.This post will go over each, and attempt to clarify why you would use them.

The Array is an ordered index of any type of items (be it an integer, some text, or a number with a period, which in ruby is called a float). It is typically denoted using [brackets] and each item is separated by commas.

i.e. cool_people = ["aneliesse", "karla", "celso", "kristina", "jessica"]

The Hash is another kind of collection of items that stresses the connection between a key, and it's value. Hashes can also take on any type of item, but it's paired inside {curly brackets} with another item using "=>"

i.e. people_i_know = { "aneliesse" => "family", "celso" => "bff", "karla" => "family" , "kristina" => "boo"}

Even though we are collecting information in both, there are instances when an array is better to use than a hash, and vice versa.

Let's say you're a Night club owner, and you were fixin' to be the hottest night spot this side of the Mississippi, so you wanted to build a program to help you out. Perhaps you wanted to make some saturday night specials and you need to collect and use information about your patrons to do so.

You decide you'd like to give the first 50 people in the door on a saturday night a free shot of whiskey! As each person comes through the door, you take their name, and you give them a shot (Don't worry officers, they're all legal drinking age). In Ruby, you'd want to use an ARRAY to collect this information. Why? because:

  • An Array prioritizes the order in which you enter information. so as you add each person's name to your array, it is counting towards 50. NOTE: in ruby, and in coding in general. when you're counting items in an array, you start with ZERO and go up from there. so the first person that comes in the door is person ZERO, not person ONE.

A few weeks go by, and you're realizing you've got some regulars that are buying their OWN shots! you want to reward them! so you decide your next special will be to give your regulars VIP treatment! Whenever they show up at the club they get their own table, and a dedicated server! Mira Que Fancy!

In ruby, you'd want to use a HASH to collect this information. Why? because:

  • A Hash doesn't worry about the order your information is put in, only the value you give it. So, your regular "Crl Crrll" comes in, and he's the 9424th person to come in? Doesn't matter, HE'S a VIP and you'll treat him as such! Hash will also let you call the values and give you a list, regardless of where the items are in the Hash.

Starting to make sense?

To summarize, Hashes and Arrays are wonderful ways to collect and manipulate information within ruby, but knowing what's most important to you, between value and order, will help you decide which path to take.
