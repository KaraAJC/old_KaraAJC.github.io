Understandng the Cycle Enumerable method in Ruby
November 2, 2014

On today's episode of Git it Gurl: the nerdy adventures of Kara Carrell; we're learning about the Cycle Method.

Huh? Method? What's that??

Let's step back and get some background info. In the Ruby programming language, everything is an object. when you want to do something to the object, you can use a "method" to do it. there are lots of methods that are defined in Ruby for anyone to use, and one of them is called the Cycle method.

the Cycle method is part of the enumerable methods, which allows you to use this method on Arrays and Hashes alike!. The way you call the Cycle method is to stick it onto an established array or hash, and give it a number of times you'd like the object to cycle through all it's elements. For example:

carrell_family = ["Carl Jr.", "Karla", "Kara", "Margie", "Carl Sr."] => Creates a new array of names as strings carrell_family.cycle(3) {|name| puts name} => Calls the cycle method to run each name and put it line by line until you have cycled through the list of names 3 times.

Why would I want to use this??

If I wanted to assign each person in my family a chore, for example, I could call Cycle to help run the list of family names and assign it an accompanying chore from another array, and make a chore chart hash.

When you utilize other related methods like .next or .rewind, you can do some pretty cool stuff. Check out this guy's super nerdy examples of how you can use .cycle.
